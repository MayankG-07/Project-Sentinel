# Project Sentinel File Structure Overview

This document outlines the file and directory structure of Project Sentinel, along with a brief description of each component.

```
.
├── .git/                               # Git version control metadata
├── .idea/                              # IntelliJ/PyCharm IDE configuration files
├── .venv/                              # Python virtual environment (created locally)
├── client/                             # Frontend application (React/Vite)
│   ├── public/                         # Static assets for the frontend
│   │   ├── rendered_pages/             # Directory for dynamically rendered PDF pages (images)
│   │   └── vite.svg                    # Default Vite logo
│   ├── src/                            # Frontend source code
│   │   ├── assets/                     # Frontend assets (e.g., images, icons)
│   │   ├── App.css                     # Main CSS for the React application
│   │   ├── App.jsx                     # Main React component, handles chat UI, authentication, and source display
│   │   ├── index.css                   # Global CSS styles
│   │   └── main.jsx                    # Entry point for the React application
│   ├── .dockerignore                   # Files/patterns to ignore when building frontend Docker image
│   ├── .eslintrc.js                    # ESLint configuration for JavaScript/React
│   ├── .gitignore                      # Files/patterns to ignore in Git for the client directory
│   ├── index.html                      # Main HTML file for the frontend application
│   ├── nginx.conf                      # Nginx configuration for serving the frontend in Docker
│   ├── package.json                    # Node.js project metadata and dependencies for the frontend
│   ├── package-lock.json               # Records the exact versions of frontend dependencies
│   ├── README.md                       # Frontend-specific README (if any)
│   └── vite.config.js                  # Vite build configuration
├── chroma_db/                          # Persistent storage for the ChromaDB vector database (generated by ingest.py)
├── Data/                               # Directory where user places PDF documents for ingestion
├── db_data/                            # Contains local SQLite database files (e.g., company.db)
├── venv/                               # Another Python virtual environment (potentially redundant or for a different setup)
├── .dockerignore                       # Files/patterns to ignore when building Docker images for the root
├── .env.example                        # Example file for environment variables (e.g., API keys, database connections)
├── .gitignore                          # Files/patterns to ignore in Git for the project root
├── auth.py                             # Handles API key authentication and user role management for the backend
├── company.db/                         # Placeholder or actual directory for a company-specific database (likely SQLite)
├── Dockerfile.backend                  # Dockerfile for building the Python backend image
├── docker-compose.yml                  # Docker Compose configuration for multi-service deployment
├── DISTRIBUTION_GUIDE.md               # Guide for distributing the application
├── encrypt_users.py                    # Script for encrypting user data (e.g., users.json)
├── forensic_audit.log                  # Log file for forensic auditing of user queries and AI interactions
├── generate_hash.py                    # Utility script to generate password hashes (e.g., for API keys)
├── generate_key.py                     # Utility script to generate API keys
├── ingest.py                           # Script for processing documents (PDFs) and ingesting them into ChromaDB
├── LICENSE                             # Project license information
├── PROJECT_STRUCTURE.md                # This file: describes the project's file structure
├── README.md                           # Main project README, providing an overview, features, tech stack, and setup instructions
├── requirements.txt                    # Python dependencies for the backend
├── sentinel.env                        # Environment variables for the backend (local configuration)
├── server.py                           # Main FastAPI backend application, handles API endpoints, AI logic, and data retrieval
└── users.json                          # (Potentially) stores user data in JSON format (might be encrypted)
```
